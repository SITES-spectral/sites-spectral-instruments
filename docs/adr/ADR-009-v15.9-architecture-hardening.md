# ADR-009: v15.9.0 Architecture Hardening

## Status

**Accepted**

## Context

A comprehensive audit (2026-03-13) of webapp-instruments v15.8.7 revealed:

- **3 critical security issues**: Token revocation race condition (`INSERT OR IGNORE` silently drops re-revocations), `CF_ACCESS_AUD` optional (allows cross-application JWT replay), and hardcoded global admin emails (operational rigidity, credential leak in source code).
- **4 DIP violations**: `ROIController`, `AnalyticsController`, `ExportController`, and `UserController` instantiated infrastructure adapters directly instead of receiving them from the DI container.
- **1 runtime TypeError risk**: `UserService.listUsers()` calls `stationRepository.findByNormalizedName()` which did not exist on `D1StationRepository`.
- **Dead code**: The versioned port infrastructure (ADR-007) was never used in production ŌĆö `UserRepositoryV1`, `UserRepositoryV2`, `UserRepositoryMigrations`, `PortVersion`, `VersionedPortAdapter`, and `versioning/index.js` were reference implementations only.
- **Domain impurity**: `UserService` contained infrastructure-specific strings (`'cloudflare-secret'`).
- **Stale version strings**: Health and info endpoints reported `15.6.0` / `15.1.0`.

The user explicitly stated: "without backward compatibility, remove dead code."

## Decision

### Security Fixes

1. **Token revocation**: Change `INSERT OR IGNORE` ŌåÆ `INSERT OR REPLACE` in `revokeToken()`. This ensures that concurrent logout + refresh cannot leave a stale token unrevoked.

2. **CF_ACCESS_AUD mandatory**: `CloudflareAccessAdapter` throws `Error('CF_ACCESS_AUD is required')` if the env var is missing. This prevents JWT replay across CF Access applications in production.

3. **Admin emails from env var**: Replace hardcoded `GLOBAL_ADMIN_EMAILS` array with `CF_ACCESS_GLOBAL_ADMINS` environment variable (comma-separated). Falls back to empty array if unset. Set via `wrangler secret put`.

### DIP Compliance

4. All 4 controllers refactored to receive dependencies from the DI container:
   - `ROIController(container, env)` ŌĆö gets `container.repositories.roi`
   - `AnalyticsController(container, env)` ŌĆö gets `container.repositories.analytics`, converted from static to instance
   - `ExportController(container, env)` ŌĆö gets `container.repositories.export`, converted from static to instance
   - `UserController(container, env)` ŌĆö gets `container.ports.credentials` and `container.repositories.station`

5. Router updated to instantiate all controllers uniformly via `new Controller(container, env)`.

### Runtime Fix

6. Add `D1StationRepository.findByNormalizedName(normalizedName)` ŌĆö queries `WHERE LOWER(normalized_name) = ?`.

### Dead Code Removal

7. Delete all versioned port infrastructure (6 files, ~1,120 lines):
   - `src/domain/user/UserRepositoryV1.js`
   - `src/domain/user/UserRepositoryV2.js`
   - `src/domain/user/UserRepositoryMigrations.js`
   - `src/domain/shared/versioning/PortVersion.js`
   - `src/domain/shared/versioning/VersionedPortAdapter.js`
   - `src/domain/shared/versioning/index.js`

8. Remove `domain/user/index.js` re-exports and `domain/shared/index.js` versioning export.

9. Remove legacy `adminRepository` alias from `container.js` (use `container.repositories.admin`).

### Domain Purity

10. Replace `created_source: 'cloudflare-secret'` ŌåÆ `'external'` and genericize management note in `UserService`.

### Deprecates

- **ADR-007** (Port Versioning Strategy) ŌĆö infrastructure removed, ADR marked deprecated.
- **`docs/PORT_VERSIONING.md`** ŌĆö moved to `docs/legacy/`.

## Consequences

### Positive

- **Security hardened**: Token revocation race eliminated, JWT replay prevented, admin emails no longer in source code.
- **DIP-compliant**: All controllers receive dependencies from container ŌĆö testable with mock repositories.
- **~1,120 lines removed**: Less surface area, no dead-code confusion.
- **Domain purity**: No infrastructure-specific strings leak into domain layer.
- **All 1,283 tests pass** ŌĆö zero regressions.

### Negative

- **`CF_ACCESS_GLOBAL_ADMINS` required**: Must be set as wrangler secret before CF Access admin login works. Without it, no user is recognized as global admin via CF Access.
- **`CF_ACCESS_AUD` required**: Deployment will fail JWT verification if env var is missing (intentional ŌĆö fail-closed).

### Neutral

- Port versioning pattern can be re-introduced if needed in the future, but the current codebase simply adds methods to existing ports instead.
